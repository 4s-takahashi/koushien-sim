# 高校野球デイズ

高校野球の監督となり、選手を育て、練習メニューを組み、大会で勝ち上がって**甲子園**を目指すシミュレーションゲームです。

🎌⚾ **ブラウザだけで遊べます。インストール不要。**

## 遊び方（5ステップ）

1. **ゲームを開始する** — 学校名・都道府県・監督名を入力してスタート
2. **チームを確認する** — チーム画面で選手一覧とラインナップを確認
3. **練習メニューを選ぶ** — ホーム画面で今日の練習内容を選択（7種類）
4. **1日ずつ or 1週間まとめて進める** — 時間を進めながら選手を成長させる
5. **大会で勝ち上がる** — 夏季大会・秋季大会を制して甲子園を目指す

さらに：
- **スカウトする** — 中学生を視察・勧誘して有望選手を獲得
- **セーブ/ロードする** — 6スロット（手動3 + 自動3）で進行を保存

## 操作説明

### ホーム画面（`/`）
- 今日の練習メニュー選択（7種類）
- 「練習して1日進む」または「1週間まとめて進む」
- チーム概要・注目選手・今後の予定を確認
- 最近のニュース（試合結果・スカウト情報）
- セーブ・ロードボタン

### チーム（`/team`）
- 選手一覧（背番号・ポジション・総合力・コンディション）
- 自動生成されたラインナップ
- 各選手の詳細ページへのリンク

### 選手詳細（`/team/[id]`）
- プロフィール（学年・ポジション・体格）
- 能力値（打撃・投球・走塁・守備）バーグラフ付き
- コンディション（疲労・怪我・精神状態）
- 通算成績

### スカウト（`/scout`）
- 中学生選手のリスト・フィルター（学年・ポジション・能力帯）
- 視察（スカウトレポート生成）
- 勧誘（獲得交渉）
- ウォッチリスト管理
- 月ごとの視察予算管理

### 試合結果（`/results`）
- 通算成績（勝敗）
- 試合カード（スコア・ハイライト）
- イニングごとのスコアボード
- 先発投手情報・打席フロー

### 大会（`/tournament`）
- 現在の大会フェーズ表示
- トーナメント表（ブラケット形式、48校シングルエリミネーション）
- 過去大会の成績履歴

### OB（`/ob`）
- 卒業生一覧（進路：プロ・大学・社会人・引退）
- 通算成績・全体統計

### セーブ・ロード
- **手動スロット 3つ**（いつでも上書き可能）
- **自動セーブ 3種**：年度替わり前・月次・大会前
  - 年度セーブは保護（上書き不可）
- データはブラウザの localStorage に保存（最大5MB）
- ストレージ使用量の表示

## 練習メニュー

| ID | 名称 | 主な効果 |
|----|------|---------|
| `batting_basic` | 基礎打撃練習 | コンタクト・技術向上 |
| `batting_live` | 実戦打撃練習 | 実戦的打撃力向上 |
| `pitching_basic` | 投球基礎練習 | コントロール向上 |
| `pitching_bullpen` | 投手ブルペン強化 | 球速・スタミナ向上 |
| `fielding_drill` | 守備練習 | 守備力・肩力向上 |
| `running` | 走塁・体力練習 | 速度・スタミナ向上 |
| `rest` | 休養（疲労回復） | 疲労大幅回復 |

夏合宿・冬合宿期間中は練習効果が 1.5 倍になります。

## 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 16 (App Router) |
| UI | React 19 + TypeScript 5 |
| 状態管理 | Zustand 5 |
| スタイル | CSS Modules（和風デザイン） |
| テスト | Vitest（505テスト / 100%パス） |
| アーキテクチャ | ViewState Projector パターン |
| データ保存 | localStorage + SHA-256 チェックサム |
| RNG | seedrandom（決定論的乱数） |

## セットアップ手順

### 必要環境

- Node.js 20.9 以上
- npm

### ローカルで動かす

```bash
# 1. リポジトリをクローン
git clone https://github.com/4s-takahashi/koushien-sim.git
cd koushien-sim

# 2. 依存関係インストール
npm install

# 3. 開発サーバー起動
npm run dev
```

ブラウザで http://localhost:3000 を開く。

### 本番ビルドの確認

```bash
npm run build
npm start
```

## テスト

```bash
# 全テスト実行（505テスト）
npx vitest run

# ウォッチモード（開発中）
npx vitest
```

テスト構成：
- `tests/engine/` — ゲームエンジン単体テスト（カレンダー・成長・試合・ワールド）
- `tests/engine/save/` — セーブ/ロード（チェックサム・破損データ検出）
- `tests/e2e/` — E2E シーズン通しテスト
- `tests/ui/` — ViewState プロジェクターテスト
- `tests/platform/` — ストレージ抽象化テスト

## デプロイ手順（Vercel）

### GitHub 連携（推奨）

1. [vercel.com](https://vercel.com) にアクセスしてログイン
2. 「Add New... → Project」→「Import Git Repository」で本リポジトリを選択
3. 設定はデフォルトのまま「Deploy」をクリック
4. 本番 URL が発行される（例: `https://koushien-sim.vercel.app`）

以降は `main` ブランチへの push で自動デプロイ。

### Vercel CLI

```bash
npm install -g vercel
vercel login
vercel --prod
```

詳細は [DEPLOY_GUIDE.md](./DEPLOY_GUIDE.md) を参照。
