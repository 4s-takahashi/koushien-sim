## 設計判断

### A1, A2: プレイボール / チェンジ後のディレイ

**現状**: ナレーション表示と次のピッチ実行が同フレームで連続している
**修正案**:
```typescript
// src/ui/match-visual/MatchPlayerHooks.tsx あたりの useEffect で
// 「━━━ プレイボール ━━━」「━━━ 3アウト・チェンジ ━━━」イベントの
// 後に setTimeout(3000) で次のピッチ実行を遅延させる
const PLAY_BALL_DELAY_MS = 3000;
const CHANGE_DELAY_MS = 3000;
```
**判断**: 「自動進行スピード設定」を尊重するため、`PLAY_BALL_DELAY_MS = 3000ms × autoSpeedMultiplier` とする
- 通常速度(x1): 3秒
- 高速(x2): 1.5秒
- 倍速(x4): 0.75秒

### A3: チャンス/ピンチでない時の自動進行停止バグ

**現状**: pendingDecision/decisionRequired 系で、不要なケースまで停止フラグが立っている可能性
**修正方針**:
- `src/engine/match/runner.ts` または `match-store.ts` を調査
- 停止理由を整理: `chance | pinch | bunt-decision | substitution | manual-pause` 以外で止まっていたら自動再開
- ログを残して「なぜ止まったか」を記録（デバッグ用）

### A4: フォアボール時の実況ログ

**現状**: フォアボール（4ボール）の特別ログが無い可能性
**修正**: `src/ui/narration/buildNarration.ts` に `walk` イベント時のログ追加
```
text: 'フォアボール！打者は一塁へ！'
icon: '🚶' (or similar)
```

### A5: 三振後の次打者登場ディレイ + ログ

**現状**: 三振直後に次打者が即登場、ログのみ
**修正**:
1. 三振イベント表示 → 1.5秒待機
2. 「次の打者: ◯◯選手（◯番、ポジション）」ログ表示
3. さらに 0.5秒待機
4. 投球開始
合計 **2秒の演出時間を挿入**

### B1, B2: ホーム画面ナビ追加

**現状**: GlobalHeader.tsx には「練習」「スタッフ」「試合」がデスクトップ幅で中央に追加されている（v0.43.0変更）
**指示**: ホーム画面の上部メインナビ（ホーム・チーム・ニュース・スカウト・大会・試合結果・OB）の列に追加してほしい
**修正方針**:
- `src/components/GlobalHeader.tsx` から「練習・スタッフ・試合」を削除
- ホーム画面のメインナビゲーション(`src/app/play/page.tsx` または同等のレイアウト) に「練習」「スタッフ」「試合」を追加
- 結果: 全10項目のメインナビ
  - ホーム / チーム / 練習 / スタッフ / ニュース / スカウト / 大会 / 試合 / 試合結果 / OB
- すべてのナビ項目にバッジ機能（現状は練習・スタッフ・試合のみ）
- ニュース: 未読件数 / スカウト: 新規候補 / 大会: 残り日数 / 試合結果: 直近結果 / OB: 新規連絡など
- バッジが不要な項目は0表示か非表示

### B3: チーム全体練習 3メニュー選択

**現状**: 1つのメニューで全体練習
**修正**: チーム練習を **3スロット** に変更
```typescript
type TeamPracticeSlot = {
  menu: PracticeMenu;
  intensity: number; // 0-100
};
type TeamPracticePlan = {
  slots: [TeamPracticeSlot, TeamPracticeSlot, TeamPracticeSlot];
  // 各スロット時間 = 全体練習時間 / 3
};
```
- 効果は各スロットの効果を 1/3 ずつ加算
- UI: 3つのプルダウン（または3列のメニュー選択）

### B6: 練習成果履歴（言葉表現）

**新規実装**:
```typescript
interface PracticeFeedback {
  date: GameDate;
  practiceType: string; // 例: 'バッティング'
  message: string;       // 例: 'ミート率があがったような気がする'
  delta: { stat: keyof Stats; value: number }; // 内部用、表示しない
}
```
- 選手詳細ページに「最近の成長」セクション
- 直近10件まで表示
- 内部で記録した stat 変化を「気がする」表現に変換するメッセージビルダー

**メッセージテンプレート例**:
- meet +2 → 「ミート率があがったような気がする」
- meet +5 → 「ミート率がしっかり上がっている」
- power +3 → 「打球が遠くまで飛ぶようになった」
- velocity +1 → 「球速がほんの少し増したかも」
- velocity +3 → 「球速の伸びを感じる」

### C1: やる気バグ修正

**現状（マギ調査済み）**:
- `src/engine/growth/motivation.ts` に `addMotivation()` 関数は存在
- しかし `src/engine/world/world-ticker.ts` で **motivation 更新が呼ばれていない**

3. `motivation.ts` の `tickMotivation()` 関数を新設して整理

### C3: 成長イベント生成

**新規実装**:
```typescript
interface GrowthEvent {
  id: string;
  playerId: string;
  date: GameDate;
  type: 'pitch_acquired' | 'opposite_field' | 'breakthrough' | 'injury_recover' | 'mental_shift';
  description: string;  // 'カーブを習得した！' 等

}
```
- 練習継続 + 適性 + 偶然要素で稀発生（基本確率 0.5%/日 等）
- 自校ニュースに自動投稿
- イベントログに永続化
