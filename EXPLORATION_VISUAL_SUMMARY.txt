🎯 KEY INSIGHTS
═════════════════════════════════════════════════════════════════════════════

ARCHITECTURE PATTERN
  User Input → Zustand Store → Game Engine → ViewState Projectors → React UI
  
  This creates clear separation:
  • Engine: Pure logic, no React dependencies
  • Store: State mutations and action dispatch
  • Projectors: Pure functions (WorldState → ViewState)
  • UI: React components using hooks

SEASON PHASES (6 per year)
  ├── 4/1–7/9    : spring_practice   (春季練習)
  ├── 7/10–7/30  : summer_tournament (夏の大会)
  ├── 7/31–9/14  : post_summer       (夏以降練習)
  ├── 9/15–10/14 : autumn_tournament (秋の大会)
  ├── 10/15–1/31 : off_season        (オフシーズン)
  └── 2/1–3/31   : pre_season        (始動期間)

TOURNAMENT SYSTEM
  • 48 schools total
  • 6 rounds (Round 1: 16 matches → Final: 1 match)
  • Seeding: Top 16 schools get bye to Round 2
  • Win probability: Reputation-based with RNG
  • Upsets: Detected when loser has 15+ higher reputation

DAY PROGRESSION TIERS
  ├── Tier 1 (Full):     Player school - detailed simulation
  ├── Tier 2 (Standard):  Other schools - batch growth
  └── Tier 3 (Minimal):   Distant schools - weekly bulk growth

MODAL IMPLEMENTATION
  Pattern: Overlay + Click-outside-to-close
  • SaveLoadPanel uses React state (local)
  • Confirmation dialogs render conditionally
  • Full-screen semi-transparent overlay
  • Nested panels with event propagation control

🔄 DATA FLOW EXAMPLES
═════════════════════════════════════════════════════════════════════════════

SINGLE DAY PROGRESSION
  1. User selects menu from dropdown
  2. User clicks "▶ 練習して1日進む"
  3. advanceDay(menuId) is called
     ├─ Gets current worldState
     ├─ Creates RNG: seed + date string
     ├─ Calls advanceWorldDay(worldState, menuId, rng)
     │  ├─ Processes player school (Tier 1)
     │  ├─ Processes other schools (Tier 2/3)
     │  ├─ Processes middle school pool
     │  ├─ Generates news
     │  └─ Returns WorldDayResult
     ├─ Accumulates in recentResults (max 30)
     ├─ Accumulates in recentNews (max 20)
     └─ set({ worldState: nextWorld })
  4. React re-renders via hook subscription
  5. getHomeView() projects new ViewState
  6. UI displays updated date, team power, news, etc.

TOURNAMENT CREATION
  1. User clicks "夏大会を開始" button
  2. startTournament('summer') is called
     ├─ Gets current worldState
     ├─ Creates bracket ID: "tournament-summer-Y-M"
     ├─ Creates RNG: seed + tournament ID
     └─ Calls createTournamentBracket(id, 'summer', year, schools, rng)
        ├─ Sorts schools by reputation
        ├─ Selects top 16 as seeds (2-round bye)
        ├─ Creates match slots for 6 rounds
        └─ Returns empty TournamentBracket
  3. set({ worldState: {..., activeTournament: bracket} })
  4. UI displays bracket with "Simulate" button

TOURNAMENT SIMULATION
  1. User clicks "▶ 大会を全試合シミュレート"
  2. simulateTournament() is called
     ├─ For each round 1-6:
     │  └─ simulateTournamentRound(bracket, roundNumber, schools, rng)
     │     ├─ For each match in round:
     │     │  ├─ Calculate home win probability
     │     │  ├─ Generate winner via RNG
     │     │  ├─ Generate scores
     │     │  └─ Detect upset
     │     └─ Return updated round
     │  └─ propagateWinners() to next round
     ├─ After round 6, determine champion
     └─ Return completed TournamentBracket
  3. set({ worldState: {..., activeTournament: completed, tournamentHistory: [...]} })
  4. UI displays completed bracket with champion
