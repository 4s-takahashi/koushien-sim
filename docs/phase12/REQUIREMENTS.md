# Phase 12「試合画面ビジュアル化」— 要件定義

**受領日:** 2026-04-21
**送信元:** 高橋さん
**目的:** 試合画面を「情報列挙」から「見るスポーツ」へ転換、プレイヤーの没入感を飛躍的に向上させる

---

## ユーザーからの要件（原文）

> もっとエキサイティングにビジュアル的にしたい。

### 要件1: スコア票の表示タイミング見直し
- **現状**: 常に画面上に表示されている
- **変更後**: 表裏の**始まりのみ**に表示、その後は隠し、**チェンジのタイミングで再表示**
- つまり、「表始まり 1回表」や「裏始まり 1回裏」の瞬間にスコア票がアニメーションで出てくる感じ

### 要件2: 野球グラウンドのビジュアル表示（鳥瞰）
**空いたスコア票のスペースに、野球グラウンドの鳥瞰ビジュアルを配置**

#### 表示要素
- フィールドのベクターイラスト（リファレンス画像: `assets/ballpark-reference.png`）
- **選手を丸で表示** — チームによって色を変える（例: 自校=青系、相手校=赤系）
- **ボールを表示** — ピッチャーが投げる、打たれると打球方向に飛ぶ
- **ボールの影を表示** — 空中を飛んでいるか地面を転がっているかが分かる
  - ボールと影の距離 → ボールの高さを視覚的に表現
- **ボールの速度・大きさ** — 速さを計算、高さを計算し、見た目の大きさを変える
  - 近く（低空）= 大きい、高空 = 小さい
- **カウント・スコアは左上にコンパクトに表示**（スコア票の代わり）
  - ストライク/ボール/アウト
  - イニング
  - 両チームの点数

#### リファレンス画像
- `assets/ballpark-reference.png`: フィールド全景のフラットデザインイラスト
- フラットデザイン、鳥瞰、鮮やかなグリーン＋ベージュ、黄色のファウルポール、グレーの観客席

### 要件3: ストライクゾーン表示と着弾点履歴
**グラウンド表示の右スペースに、ストライクゾーンを表示**

#### 表示要素
- 3×3 のグリッド（ストライクゾーン）
- リファレンス画像: `assets/strike-zone-reference.png`
- 投手視点（キャッチャー後方カメラ）
- **各球の着弾点を表示**
  - **ストレート = ◯（円マーカー）**
  - **変化球 = 二等辺三角形**、**頂点を変化の方向に向けて表示**
    - 例: 内角に曲がるスライダー → 三角形の頂点が内側向き
    - 例: 落ちるフォーク → 三角形の頂点が下向き
- **バットを振った位置も表示**
- **マーカー履歴**: 1打席が終わるまで着弾点は残す、**次のバッターになったらクリア**
- **色分け**: ストライク=赤系、ボール=緑系（リファレンス参考）
- **番号表示**: 投球順を番号で表示（①〜⑤）

## 実装時の留意点（画像分析結果より）

### サイズ・レイアウト
- ストライクゾーンは画面の 30〜40% を占める中央配置が見やすい
- グリッドの1マスは最低 48×48px 以上
- 鳥瞰フィールドは**メインビュー**として大きく表示（副ビューではない、この Phase では主役）

### 座標系
- ストライクゾーン: ローカル座標 (0,0)〜(1,1) の正規化UV座標
- ゾーン中心を (0.5, 0.5)、外れ玉は範囲外（±0.3 マージン）
- フィールド鳥瞰: ホームベースを原点にした極座標 or ワールドXZ座標
- 打球落下点の変換関数（3Dワールド → 2Dミニマップ）

### アニメーション候補
| シーン | アニメーション案 |
|---|---|
| イニング開始 | **スコア票スライドイン → 2秒表示 → スライドアウト** |
| 投球時のマーカー出現 | スケールイン + フェードイン（0.2s） |
| ストライク判定 | 赤マーカーにリング状パルス、グリッドマス点滅 |
| ボール判定 | 緑マーカーに軽いバウンド |
| カウント変化 | 画面左上のカウント表示を数字フリップ |
| 打球発生時 | 鳥瞰フィールドで打球曲線を Bezier で描画、着地点にリング拡散 |
| ホームラン | ファウルポール発光 + スタンドへの軌跡アニメ |
| ボールの影 | ボールの真下にぼやけた楕円、高さに応じて大小変化 |

### 実装Tips
- マーカーは履歴スタック（最大5〜10球）で管理、古いものは透明度を下げる
- 色覚多様性対応: 赤緑だけでなく形状（◯/△）や番号で区別
- グリッドは半透明（α=0.6程度）で背景が透けて見える
- UIレイヤーは HUD層（固定）と 3Dビュー層（可変）を分離

## 技術選択肢

### 選択肢A: Canvas 2D API
- ブラウザ標準、外部ライブラリ不要
- パフォーマンス良好（60fps 実現可能）
- React との統合は refs で
- **推奨**（軽量、依存ゼロ）

### 選択肢B: SVG + CSS animation
- React との統合が直感的
- 数十個のマーカー・選手マーカー程度なら問題なし
- 打球軌跡や影はやや工夫が必要
- **アニメーションの柔軟性は Canvas より低い**が、DOM でデバッグしやすい

### 選択肢C: Pixi.js / Phaser
- ゲームエンジン、リッチな表現可能
- バンドルサイズ大（Pixi: 430KB+）
- オーバースペックの可能性

### 選択肢D: Three.js （3D）
- 本格的な3D表現
- 学習コスト高、パフォーマンス懸念
- この要件には**不要**（鳥瞰2Dで十分）

**推奨**: **A (Canvas 2D)** でフィールド・ボール・軌跡 + **B (SVG)** でストライクゾーン・マーカー UI
- 理由: フィールドは多数のオブジェクト＋アニメが走るので Canvas が有利、ストライクゾーンは静的レイアウト＋HTML親和性が必要なので SVG が扱いやすい

## データモデルの拡張

### Pitch（投球）データ
```typescript
interface Pitch {
  // 既存
  result: PitchResult;  // ball/strike/hit
  count: { balls: number; strikes: number; outs: number };

  // 🆕 追加
  pitchType: 'fastball' | 'slider' | 'curve' | 'forkball' | ...;
  location: { x: number; y: number };  // ストライクゾーン内の座標 (0-1)
  velocity: number;  // km/h
  breakDirection?: { dx: number; dy: number };  // 変化球の変化方向
  swingLocation?: { x: number; y: number };  // スイングした位置
}
```

### BallAnimation（打球）データ
```typescript
interface BallTrajectory {
  startPos: { x: number; y: number };  // ホームベース
  endPos: { x: number; y: number };    // 着弾点
  peakHeight: number;                   // 最高到達点
  durationMs: number;                   // 飛行時間
  type: 'fly' | 'grounder' | 'linedrive';
}
```
