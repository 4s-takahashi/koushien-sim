# Phase 12「試合画面ビジュアル化」— 全体方針・UIレイアウト

**作成日:** 2026-04-21
**状態:** 設計完了・実装待ち
**差し込み位置:** Phase 11.5（物語化リデザイン）の次
**依存フェーズ:** Phase 10-B/C（試合画面UI）完了済み

---

## 1. 目的・背景

### なぜこの変更か

Phase 10 で「1球1球の采配」が実現した。現状の試合画面は **情報として正確だが、見ているだけで興奮しない**。
高橋さんの言葉を借りれば「もっとエキサイティングにビジュアル的にしたい」。

現在の課題:
- スコア票が常時表示 → イニングの節目感がゼロ
- ダイヤモンド表示はアイコンのみ → 「球場にいる感」がない
- 投球の着弾点は5×5グリッドの番号だけ → 配球の読み合いが伝わらない
- 打球がどこに飛んだか視覚情報がない → 守備側の采配機会が生まれない

### Phase 12 が実現する体験

> **「テキスト観戦」から「球場観戦の疑似体験」へ**

1. **スコア票** — 表裏の始まりにのみスライドイン、2秒後に自動退場。節目感が生まれる
2. **グラウンド鳥瞰** — 選手とボールが動くフィールドビュー。「今どの守備位置にいるか」が一目瞭然
3. **ストライクゾーン** — 配球履歴を◯/△で可視化。投手の攻め方・打者の弱点が読めるようになる

## 2. Phase 11.5「物語化リデザイン」との関係

Phase 11.5 では「言葉評価」による選手の物語化を実装する。Phase 12 のビジュアル化と組み合わせることで、
試合画面上に**言葉（ドラマ）× ビジュアル（映像）**の両方が揃う。

### 具体的な共存ポイント

| 要素 | Phase 11.5 の寄与 | Phase 12 の寄与 |
|---|---|---|
| 投手プロフィールポップアップ | 能力の「言葉評価」（「制球が精密」等） | スタミナバー・球種バッジ |
| 打者プロフィールポップアップ | 打撃傾向の言葉表現 | 打率・調子・特性バッジ |
| 実況ログ（Phase 7-A） | 続投・採用（変更不要） | ストライクゾーンへのマーカー追記で連動 |
| 左上HUD（カウント/スコア） | 言葉ラベルは Phase 11.5 の翻訳関数を流用 | HUD の表示エリアを Phase 12-C で実装 |

### 実装時の連携指針

- `matchProjector.ts` への視覚情報追加は **Phase 12-A で一度だけ** 行い、Phase 11.5 のビュー型変更と衝突しない設計とする
- 投手・打者パネルのポップアップ（`PsycheWindow` / 詳細采配モーダル）は Phase 11.5 の言葉評価を後から差し込める枠を確保しておく

## 4. Phase 分割と全体フロー

```
Phase 12-A（基盤整備）
  ├─ レンダラー基盤（Canvas Renderer / SVG 基盤クラス）
  ├─ スコア票 表示タイミング制御（イニング始まりのみ）
  └─ ストライクゾーンUI（静的、マーカーなし）

Phase 12-B（ストライクゾーン実装）
  ├─ 投球ごとにマーカー描画（◯/△）
  ├─ 変化球の方向を三角形の向きで表現
  ├─ バットスイング位置表示
  └─ 打席間クリア + アニメーション

Phase 12-C（グラウンド表示）
  ├─ フィールドSVG/Canvas描画
  ├─ 選手マーカー9人（チーム別色分け）
  └─ 左上HUD（カウント/スコア/イニング）

Phase 12-D（ボール・打球アニメーション）
  ├─ 投球時ボール移動（マウンド→捕手）
  ├─ 打球軌跡 Bezier 曲線
  └─ ボール影（高さ表現）

Phase 12-E（演出強化）
  ├─ イニング開始スコア票スライドイン
  ├─ ホームラン演出（ファウルポール発光）
  └─ パフォーマンス最適化（60fps検証）
```

## 5. 設計原則の確認

| 原則 | 実現方法 |
|---|---|
| エンジン無変更 | `src/engine/match/` は一切触らない。視覚情報は投影層（projector）で派生させる |
| 段階的ロールアウト | Phase 12-A〜E を独立してリリースできる。各サブフェーズ完了時点でデプロイ可能 |
| パフォーマンス最優先 | Canvas 2D + requestAnimationFrame。スマホ 60fps 維持 |
| アクセシビリティ | ◯/△の形状区別 + 番号 + 色（3重冗長）。`prefers-reduced-motion` 対応 |
| レスポンシブ | CSS Grid + resizeObserver でキャンバスサイズを動的計算 |
| テスト可能性 | 描画ロジックは pure function 化。Canvas mock で単体テスト可能 |
| i18n | テキスト要素（HUDラベル等）は `src/ui/labels/` の既存パターンを踏襲 |

## 7. リスク一覧（全体）

| リスク | 確率 | 影響度 | 緩和策 |
|---|---|---|---|
| スマホ 60fps 維持できない | 中 | 高 | Phase 12-A で Canvas プロトタイプ → fps 計測してから先に進む |
| 既存テスト破損 | 低 | 高 | engine/ は無変更。projector 変更は後方互換で型追加のみ |
| SVG/Canvas レイアウト崩れ | 中 | 中 | ResizeObserver + aspect-ratio CSS で対応 |
| Phase 11.5 との型衝突 | 低 | 中 | view-state-types.ts 変更は optional フィールドのみ追加 |
| デザイナーなしでのビジュアル品質 | 高 | 中 | フラットデザイン＋固定カラーパレット。ballpark-reference.png を参考に |
