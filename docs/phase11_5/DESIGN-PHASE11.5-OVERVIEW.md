# Phase 11.5「物語化リデザイン」— 全体方針と変更点サマリ

**作成日:** 2026-04-21
**状態:** 設計完了・実装待ち
**差し込み位置:** Phase 11-A 完了後、Phase 11-D（磨き込み）の前
**実装順序（改訂版）:** A → **11.5** → D → B → C

---

## 1. 目的・背景

### なぜこの変更が必要か

Phase 11-A（体験深化）まで実装した結果、ゲームの骨格は整ったが、**「数値を見ているゲーム」から抜け出せていない**という課題が顕在化した。

現状の問題点:
1. **ホーム画面が作業場** — 練習メニューを選ぶ操作画面に過ぎない。チームに起きていることが見えない
2. **選手が数値の塊** — ミート 72、パワー 85 という数字は、監督の「感情」を動かさない
3. **評価者の不在** — 選手が注目されているのか、埋もれているのか分からない。物語的緊張感がない
4. **マネージャーが機能していない** — 対戦相手の情報収集という役割が曖昧で、スカウト機能との差別化がない

### 変更後の方向性

> **「数値のシミュレーター」から「選手たちの物語を読む体験」へ**

- ホーム画面は「今チームで何が起きているか」を伝える情報ハブになる
- 選手の能力は「言葉」で語られる（内部では数値を保持し続ける）
- 評価者（メディア・批評家・スカウト）が各選手を注目することで、選手に「社会的存在感」が生まれる
- マネージャーを通じた情報収集に深みが生まれ、戦略的な試合準備が可能になる

## 2. 変更点サマリ

| 領域 | 現状 | 変更後 |
|---|---|---|
| ホーム画面 | 練習メニュー選択画面 | チームニュースハブ（タブ切替）|
| チーム画面 | 選手一覧のみ | 練習メニュー設定 + 一括休養 |
| 選手数値表示 | 数値直接表示（ミート 72） | 言葉で表示（「鋭いミートセンス」） |
| 評価者 | なし | メディア4社・批評家10人・スカウト10人 |
| 選手プロフィール | 基本情報 + 成績 | + 悩み/心境/イベント履歴/練習履歴 |
| 対戦相手評価 | スカウトレポート（テキスト1件） | マネージャー経由の言語評価（精度可変）|
| マネージャー管理 | 単一マネージャー（名前のみ） | 育成/複数雇用/統合管理（3案から選択）|

## 3. 設計原則

1. **既存の数値ロジックは絶対に壊さない**
   - 内部では `player.stats.*` の数値を保持し続ける
   - 変更するのは「表示層のみ」（プロジェクター → ViewState → UI）

2. **段階的移行可能にする**
   - 言葉化は能力カテゴリ単位で切り替えられる設計（feature flag 的）
   - MVP は主要5能力のみ言葉化し、残りは後続サブフェーズで対応

3. **テスト可能性**
   - 言葉化関数は pure function として切り出し、unit test 可能にする
   - `src/ui/labels/ability-narrative.ts` に全集約

4. **i18n 対応**
   - 言葉プールは `src/ui/labels/` 配下の定数ファイルに集約
   - 将来の翻訳や差し替えを考慮した構造

5. **拡張性**
   - 評価者は初期24人だが、後から追加しやすいレジストリ方式
   - 言葉プールは配列で管理し、増やすだけで反映される

---

## 4. 期待効果

- **没入感向上**: 「筋力 85」より「並を超える力強さがある」の方が、選手への愛着が生まれる
- **戦略的深み**: マネージャーのランクによって対戦相手の見え方が変わる → 育成モチベーションが上がる
- **社会的緊張感**: 批評家やスカウトが選手を評価していることで、「誰かに見られている」感覚が生まれる
- **ニュース読み物化**: ホーム画面が「今日はどんなことが起きたか」を読む場所になり、毎日の進行が楽しくなる

## 5. Phase 11 全体への影響

```
Phase 11-A（完了）
  ↓
Phase 11.5（本フェーズ）
  ├─ 11.5-A: ホーム画面再設計
  ├─ 11.5-B: チーム画面・練習メニュー改善
  ├─ 11.5-C: 評価者システム基盤
  ├─ 11.5-D: 選手評価言葉化（MVP）
  ├─ 11.5-E: 選手プロフィール拡充
  ├─ 11.5-F: 対戦相手スカウティング言葉化
  └─ 11.5-G: マネージャー管理（3案から実装）
  ↓
Phase 11-D（磨き込み）← 11.5 のUIをポリッシュする機会
  ↓
Phase 11-B（データ・記録）← 評価者ランキングと連携できる
  ↓
Phase 11-C（甲子園）← 甲子園での評価者注目度が盛り上がりを加速する
```

### Phase 11-B への影響
- `DESIGN-PHASE11.5-EVALUATORS.md` で定義する評価者ランクデータは、Phase 11-B の「年度別ランキング」と連携できる
- 評価者が選手に付けたランクを記録として残せば、「あの批評家に注目された年」がログに残る

### Phase 11-C への影響
- 甲子園出場校・選手への評価者注目度が増す仕組みを組み込むと、甲子園シーズンの盛り上がりが増大する
- スカウトが甲子園で選手を評価するイベントを将来追加できる土台になる

| リスク | 内容 | 対策 |

| セーブデータ互換 | 評価者データ・イベント履歴等の追加 | optional フィールドで追加、旧セーブデータは空配列/undefined として扱う |
